          <StepHikeCompact.Details title="Verify that psql is working" fullWidth>

              Close and re-open your shell/terminal and run the following command:

              <StepHikeCompact.Code>
                ```sql
                  psql --version
                ```
              </StepHikeCompact.Code>

              If you get any error messages that psql is not available or cannot be found then the PATH variable is likely either not correctly set or you need to restart your shell/terminal.

              You can add the postgres installation path to your PATH variable by running the following to get the path for postgresql:

              ```sql
                brew info postgresql@17
              ```

              You should get a response that looks like this:

              ```
                If you need to have postgresql@17 first in your PATH, run:

                echo 'export PATH="/opt/homebrew/opt/postgresql@17/bin:$PATH"' >> ~/.zshrc
              ```

Run the command it shows and then restart the terminal. You should now be able call psql from the terminal successfully.
